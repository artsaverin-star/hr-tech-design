# HR TECH DESIGN

Мост между Figma и Claude: пишешь задачу обычными словами в терминале — Claude собирает мобильные
версии экранов из компонентов вашей дизайн-системы HRDS, исправляет орфографию, собирает сценарии,
приводит макеты к ДС. Claude знает вашу ДС, потому что база знаний (правила + справочник компонентов)
лежит в этом репозитории и подтягивается в каждую сессию.

## Как это устроено (простыми словами)

Claude — как сотрудник, у которого три части:
- **Мозг (Claude)** — нейросеть, она думает. Про вашу ДС в голове ничего нет.
- **Методичка (знания)** — текстовые файлы в этой папке (`CLAUDE.md`, `claude/commands/`). Claude читает их
  автоматически, когда ты запускаешь сессию **внутри этой папки**.
- **Руки (мост)** — плагин в Figma + MCP-сервер. Даёт Claude возможность читать и менять твой макет.

Цепочка: открыл терминал **в папке проекта** → `claude` сам прочитал методичку → нажал в плагине
**Activate Bridge** (дал руки) → пишешь задачу. Знания идут из файлов папки, НЕ из моста.

## Установка (один раз)

```bash
# 1. Node.js — поставь LTS с https://nodejs.org
# 2. Claude Code и вход в СВОЙ аккаунт:
npm install -g @anthropic-ai/claude-code
claude            # при первом запуске войди в свой аккаунт Claude

# 3. Клонируй этот репозиторий и запусти установщик:
git clone <URL-этого-репозитория> ~/projects/hr-plugin
cd ~/projects/hr-plugin
./setup.sh        # соберёт мост, подключит MCP и команды (симлинком на репо)
```

4. **Плагин в Figma** (один раз): десктопная Figma → меню → Plugins → Development →
   Import plugin from manifest… → выбери `figma-desktop-bridge/manifest.json`.

## Как работать (каждый день)

1. Открой рабочий файл в Figma, запусти плагин **HR TECH DESIGN**, нажми **Activate Bridge** — лампочка зелёная.
2. В терминале запусти Claude **из папки проекта** (важно — отсюда подтягивается методичка):
   ```bash
   cd ~/projects/hr-plugin && claude
   ```
   Нужна другая модель — добавь флаг, например для сложного экрана:
   ```bash
   cd ~/projects/hr-plugin && claude --model fable
   ```
3. Выдели нужный фрейм(ы) в Figma.
4. Опиши задачу словами:
   - «сделай мобильную версию выделенного экрана»
   - «исправь орфографию в этом фрейме»
   - «собери выделенные экраны в сценарий»
   - «приведи макет к компонентам HRDS»
   - «что ты знаешь про HRDS Drawer?» — справка по ДС
   - «добавь паттерн: …» — научить систему новому типу экрана

## Лайфхаки

- **Модель** прямо в сессии: команда `/model`. Haiku — быстро (орфография, мелочи), Sonnet — конвертации,
  Opus / Fable — сложные новые типы экранов (медленнее, качественнее). Или флаг `--model fable` при старте.
- Прервать ответ — **Esc**.
- Всегда выделяй фрейм **до** запроса.
- Проси «покажи diff» или «decision log» — проверить, что контент перенесён точно.
- Большой экран дроби на части.
- Мост (зелёная лампочка) держи активным всё время работы.

## Если не работает

- Лампочка не зелёная → проверь, что `claude` запущен, нажми Activate Bridge.
- «нет инструмента figma_execute» → перезапусти `claude`.
- Плагин завис → закрой и открой заново.

## Как дополнять знания

База знаний — единственный источник правды, лежит в репозитории:
- `claude/commands/hrtech.md` — правила сборки (zone-graph алгоритм, blueprints).
- `claude/commands/hrds-knowledge.md` — справочник компонентов и паттернов HRDS (ключи + правила).

Правишь эти файлы → `git push` → у всех обновляется через `git pull`. Подробнее — `CLAUDE.md`.
Процесс контрибуции в саму HRDS — `HRDS-CONTRIBUTION.md`.
