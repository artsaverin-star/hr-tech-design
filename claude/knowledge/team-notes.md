# База знаний команды (HR TECH DESIGN)

Здесь копятся **проверенные приёмы, ключи компонентов и фиксы**, которыми делятся дизайнеры
через кнопку **«Поделиться знанием»** в плагине. Файл версионируется в репозитории и
автоматически подтягивается в контекст Claude (через `CLAUDE.md`), поэтому знание одного
дизайнера становится доступно всем после **«Обновить базу знаний»**.

Формат: каждая заметка — отдельная секция `## Автор — дата` с текстом. Не удаляй чужие заметки;
владелец ДС периодически дистиллирует ценное отсюда в `claude/commands/hrds-knowledge.md`.

## ДС-команда — 2026-06-25 (дистилляция наработок)

**Иконка-кнопка по ДС.** Делай как `❇️ Button` с `Variant=Icon` (`Size=M/L`, `View=Ghost`; `Secondary` = ховер-фон), а саму иконку ставь через свойство-слот `← Icon#29143:237` (instance-swap) — НЕ кидай голый вектор. Тултип кнопки — через `← Label#19437:14`. Ключи иконок: шестерёнка `Navigation / Settings` `720affbd7625eed97784d060ad4b04782ed726dd`, звонок `Actions / Call` `0d2948d7ba226c7edcf8f88a39a5274856aaa3d6` (вариант `Size=16, Filled=false`).

**List-Item.** Заголовок/описание задавай, записывая ТЕКСТ-узлы `Label`/`Caption` напрямую — свойство `← Title` часто не применяется (остаётся плейсхолдер «Label»). Для «число крупно / тип мелко» включи `Reserse=on` на вложенном `Text · Leading` (отдельного перевёрнутого варианта нет).

**Состояния Button.** Дизейбл/лоадинг — это реальные `State=Disabled` / `State=Loading`, а не `opacity 0.4`.

**Токены поверхностей (контринтуитивно).** `surface/0` = серый #F5F5F5 (подложка / утопленная карточка), `surface/100` и `surface/200` = белые. Серую плашку на белом листе бинди к `surface/0`. Общего токена скругления в ДС нет — радиус кастомной поверхности оставляй литералом.

**Drawer [mobile] сборка.** Media-слот прячь (`.visible=false`); кастомный контент шторки — instance-swap, поэтому делай его отдельным КОМПОНЕНТОМ и подставляй; футер — Button · Group. Standalone Pull-Bar схлопывается в высоту 0 — рисуй ручку сам (36×4, r2, #D1D1D6).
