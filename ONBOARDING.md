# HR TECH DESIGN — установка для дизайнера

Мост между Figma и Claude: жмёшь кнопки в плагине — Claude выполняет работу по зашитым правилам (конвертация в мобилку, орфография, сборка сценариев, приведение к ДС).

У каждого дизайнера — **свой аккаунт Claude** (свои лимиты, своя подписка). Ничего общего настраивать не нужно.

## Установка (один раз, ~5 минут)

1. **Node.js** — если нет: [nodejs.org](https://nodejs.org), кнопка LTS.
2. **Claude Code** — в Терминале:
   ```
   npm install -g @anthropic-ai/claude-code
   claude
   ```
   При первом запуске войди в **свой** аккаунт Claude.
3. **Этот проект** — получи папку `hr-plugin` (git clone или копия) и запусти:
   ```
   cd hr-plugin
   ./setup.sh
   ```
   Скрипт сам соберёт сервер, подключит его к Claude Code и поставит команду `/hrtech`.
4. **Плагин в Figma** (один раз): десктопная Figma → меню → **Plugins → Development → Import plugin from manifest…** → выбери `hr-plugin/figma-desktop-bridge/manifest.json`.

## Как пользоваться (каждый день)

1. Открой рабочий файл в Figma и запусти плагин **HR TECH DESIGN** (Plugins → Development).
2. В Терминале запусти `claude` (из любой папки).
3. В плагине нажми **Activate Bridge** — лампочка должна стать зелёной (Connected).
4. Выдели нужные фреймы и жми кнопку действия:
   - **Desktop → Mobile** — собрать мобильную версию выделенных десктоп-экранов
   - **Fix Spelling** — исправить орфографию в текстах
   - **Assemble Scenario** — собрать выделенные экраны в сценарий
   - **Apply Design System** — привести макет к компонентам ДС
   Можно нажать несколько раз — задачи копятся в очередь.
5. В Claude напиши **`/hrtech`** — он разберёт очередь и отчитается, что сделал.

## База знаний команды (общие наработки)

Кроме зашитых правил, у команды есть **общая база знаний** — проверенные приёмы, ключи компонентов и фиксы, которые копятся по мере работы. Она лежит в репозитории и автоматически попадает в контекст Claude. Две кнопки в плагине:

- **🔄 Обновить базу знаний** — подтягивает последнюю версию (наработки коллег). Жми в начале работы.
- **➕ Поделиться знанием** — узнал приём / ключ компонента / фикс → впиши имя и короткий текст → **Отправить команде**. Заметка уходит в общую базу; после того как коллеги нажмут «Обновить», её видят все (и Claude учитывает её в работе).

Нужен активный мост (зелёная лампочка) и запущенный авто-раннер — результат придёт уведомлением в Figma.

**Доступ к общей базе (один раз, только чтобы ДЕЛИТЬСЯ):** «Поделиться знанием» пишет напрямую в общий репозиторий, поэтому нужен доступ на запись:
1. Владелец репозитория добавляет тебя в **Collaborators** на GitHub (один раз).
2. Ты выполняешь в Терминале `gh auth login` (или ставишь GitHub CLI: `brew install gh`) — один раз.

Кнопка **«Обновить базу знаний»** (только чтение) работает и без этого.

### Режим «нажал — само работает»

Вместо ручного `/hrtech` можно запустить авто-раннер и свернуть терминал:
```
./scripts/hrtech-watch.sh
```
Он следит за очередью: нажал кнопку в плагине → через несколько секунд Claude сам начинает выполнять задачу (от твоего аккаунта). Остановить — Ctrl+C.

## Если что-то не так

- **Лампочка не зеленеет** — проверь, что `claude` запущен в терминале; нажми Activate Bridge ещё раз; крайний случай — ссылка Console внизу плагина → Reconnect.
- **«No MCP server found»** — закрой все сессии claude и запусти одну заново.
- **Claude пишет, что нет инструмента figma_execute** — сессия запущена раньше установки; перезапусти `claude`.
- Плагин должен быть **открыт** всё время, пока Claude работает с файлом.
