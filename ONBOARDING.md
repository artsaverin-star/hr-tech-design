# HR TECH DESIGN — установка для дизайнера

Мост между Figma и Claude: запускаешь плагин, открываешь `claude` в Терминале и **описываешь задачу
словами** — Claude делает её по зашитым правилам вашей ДС (мобильная версия, орфография, сборка
сценариев, приведение к HRDS, справка по компонентам).

У каждого дизайнера — **свой аккаунт Claude** (свои лимиты, своя подписка). Общего настраивать нечего.

## Установка — одна команда (~5 минут)

Вставь **одну строку** в Терминал на Mac и нажми Enter:

```
curl -fsSL https://raw.githubusercontent.com/artsaverin-star/hr-tech-design/main/install.sh | bash
```

Она сама поставит Node.js, Claude Code, скачает проект в `~/hr-tech-design`, подключит мост и команды
`/hrtech` + `/hrtech-digest` + базу знаний, и откроет в Finder папку с плагином.

> Если Mac впервые без инструментов разработчика — установщик попросит подтвердить установку
> Xcode Command Line Tools и может спросить пароль Mac (для Homebrew). Это нормально, один раз.

Дальше — **два разовых шага руками** (их командой не сделать):

1. **Войди в Claude** (свой аккаунт): в Терминале набери `claude` и залогинься.
2. **Поставь плагин в Figma:** десктопная Figma → **Plugins → Development → Import plugin from manifest…**
   → выбери `~/hr-tech-design/figma-desktop-bridge/manifest.json`.

## Как работать (каждый день)

1. Открой рабочий файл в Figma и запусти плагин **HR TECH DESIGN** (Plugins → Development).
   Мост поднимается сам — **лампочка в шапке станет зелёной** (Connected). Отдельную кнопку жать не нужно.
2. В Терминале: `cd ~/hr-tech-design && claude` (для сложного нового экрана — `claude --model fable`).
3. Выдели нужный фрейм(ы) в Figma.
4. **Опиши задачу обычными словами.** Примеры:
   - «сделай мобильную версию выделенного экрана»
   - «исправь орфографию в этом фрейме»
   - «собери выделенные экраны в сценарий»
   - «приведи макет к компонентам HRDS»
   - «что ты знаешь про HRDS Drawer?» — справка по ДС
   - «добавь паттерн: …» — доучить систему новому типу экрана

Прогресс, аккаунт и остаток лимитов видно прямо в плагине. Прервать ответ — **Esc**.
Большой экран дроби на части. Плагин держи открытым, пока Claude работает с файлом.

## База знаний команды

В плагине две кнопки:

- **🔄 Обновить** — подтянуть наработки коллег (жми в начале работы).
- **➕ Поделиться** — узнал приём / ключ компонента / фикс: напиши как есть и отправь.
  Имя подставится само (из аккаунта Figma), формулировки причешут позже. Заметка попадёт в общую базу
  и в контекст Claude у всех после «Обновить».

**Доступ для кнопки «Поделиться» (один раз, только чтобы ДЕЛИТЬСЯ):** она пишет в общий репозиторий,
поэтому нужен доступ на запись:
1. Владелец репозитория добавляет тебя в **Collaborators** на GitHub.
2. Ты выполняешь в Терминале `gh auth login` (поставить CLI: `brew install gh`).

Кнопка **«Обновить»** (только чтение) работает и без этого.

### Режим «нажал — само работает»

Вместо ручного `/hrtech` можно запустить авто-раннер и свернуть терминал:
```
cd ~/hr-tech-design && ./scripts/hrtech-watch.sh
```
Он следит за очередью задач из плагина и запускает Claude сам (от твоего аккаунта). Остановить — Ctrl+C.

## Если что-то не так

- **Лампочка не зелёная** — проверь, что `claude` запущен в Терминале; крайний случай — ссылка
  **Console** внизу плагина → Reconnect.
- **«No MCP server found» / «нет инструмента figma_execute»** — закрой все сессии `claude` и запусти
  одну заново (сессия была запущена раньше установки).
- **Плагин завис** — закрой и открой заново. Плагин должен быть открыт всё время работы.

## Обновиться до свежей версии

```
cd ~/hr-tech-design && git pull
```
Затем переоткрой плагин в Figma (подхватит новый код виджета). Версия видна в шапке плагина.
